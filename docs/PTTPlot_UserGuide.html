1Overview

PTTPlot is the rendering engine of the PTTimeline suite. It reads .pttd timeline data files produced by PTTEdit and renders interactive Gantt-style horizontal bar charts using matplotlib embedded in a PySide6 window.

Charts can be explored interactively — zoomed, panned, annotated with task info popups, and decorated with dependency arrows — then exported to PNG, JPG, PDF, or SVG for sharing or archiving. Exported images can be opened directly in PTTView for further zoom and inspection.

Typical Workflow

  1. Use PTTEdit to create or edit a .pttd timeline data file.
  2. Open the file in PTTPlot via File → Open, or pass it on the command line.
  3. Adjust presentation options, inspect dependency arrows, and annotate tasks as needed.
  4. Export the chart via File → Save to PNG, JPG, PDF, or SVG.
  5. View the exported image in PTTView by choosing Save & View from the save dialog.

Command-Line Usage

PTTPlot can be launched with or without a file argument:

python pttplot.py
python pttplot.py path/to/file.pttd

When a filename is provided, the chart renders immediately on startup. Without a filename, PTTPlot opens to a blank window and waits for a file to be opened via the menu.

Opening from Windows Explorer

On Windows, the PTTimeline installer registers PTTPlot as an additional handler for .pttd files. Double-clicking a .pttd file opens it in PTTEdit (the default). To open it directly in PTTPlot instead, right-click the file in Explorer and choose Open with PTTPlot from the context menu or the Open with flyout.

2Interface Overview

The main window has four main areas:

Menu Bar

Three menus are available: File, Presentation, and Help. All major operations are accessible from the menu bar.

Chart Canvas

The central area displays the matplotlib canvas containing the rendered Gantt chart. Process names appear on the vertical axis. The horizontal axis represents time. Task bars are drawn as horizontal rectangles, color-coded by process.

Each task bar is interactive — right-clicking on a bar shows a task info annotation, and Ctrl-clicking reveals dependency arrows. See Click Controls for the full reference.

Zoom Controls Toolbar

A matplotlib navigation toolbar can optionally be shown below the menu bar. It is hidden by default. Enable it via Help → Zoom Controls. When visible, it enables mouse-wheel zoom and pan. See Zoom Controls Toolbar.

Status Feedback

Save progress and completion are communicated through window title changes. During a save operation the title updates to reflect the current state, then returns to normal on completion.

File Watcher: When a .pttd file is loaded, PTTPlot continuously monitors it for external changes. If the file is modified by another application (for example, by saving from PTTEdit), the chart automatically reloads and re-renders. This behaviour can be toggled on or off using Presentation → Re-Plot On PTTD Change. See Auto-Reload.

3File Operations

All file operations are available under the File menu.

Open

Opens a standard file browser to select a .pttd file. Once selected, the chart renders immediately. Any previously loaded file is replaced.

Open Recent

Opens a submenu listing the most recently opened .pttd files (up to 15 entries, newest first). Entries 1–9 have keyboard accelerators. Hovering over an entry shows the full path in a tooltip. Stale entries (files no longer on disk) are silently pruned when the submenu opens. A Clear Recent Files item at the bottom clears the entire list. The file dialog for Open defaults to the directory of the most recent entry.

Save Formats

Four export formats are available under File → Save:

Format Extension Resolution Output Size Tight Crop Notes
PNG .png 300 DPI 16.5 × 10.5 in Yes Lossless raster; best for on-screen use
JPEG .jpg 300 DPI 16.5 × 10.5 in Yes Compressed raster; smaller file size
PDF .pdf 300 DPI 16.5 × 10.5 in Yes Print-ready; embeds vector text and lines
SVG .svg N/A (vector) N/A No Fully scalable vector; DPI setting is not applied
SVG note: SVG exports do not use tight cropping. The bbox_inches='tight' parameter is applied for PNG, JPG, and PDF only. Using tight cropping with SVG in Qt's QSvgWidget causes rendering failures, so it is intentionally skipped for that format.

The Save Dialog

PTTPlot uses a custom save dialog (not the native OS file picker) that provides three buttons:

  • Save — Writes the file to disk and closes the dialog.
  • Save & View — Saves the file, then opens the exported image in PTTView (or the configured external viewer for PDF and SVG).
  • Cancel — Closes the dialog without saving.

Zoom State During Export

Exports always render the full chart (zoomed out to show all data), regardless of the current zoom or pan level. After the export completes, the canvas is restored to the zoom state it was in before saving.

Exit

Closes the PTTPlot application. No unsaved chart data is lost — the chart is regenerated from the source .pttd file each time it is loaded.

4Presentation Menu

The Presentation menu controls how task bars and labels are displayed, and provides commands for loading and saving named presentation configurations. Changes to layout options take effect immediately, re-rendering the chart.

Open…

Opens a dialog listing all .pttp presentation files found in the same directory as the currently loaded .pttd file. Presentation files are named using a dot-qualifier convention:

  • sample.{}.pttp — the default (unnamed) presentation
  • sample.{overview}.pttp — a named presentation called overview
  • sample.{detail}.pttp — a named presentation called detail

Selecting a presentation from the list reloads the chart with that presentation's configuration. The current presentation name is highlighted in the list so you can see which one is active. This item is disabled until a file is loaded.

Open Recent

Opens a submenu listing the most recently opened .pttp presentation files (up to 15 entries, newest first). This list is separate from the File → Open Recent list for .pttd files. Hovering over an entry shows the full path in a tooltip. A Clear Recent Files item at the bottom clears the list.

Save…

Opens a dialog where you can type a presentation name (alphanumeric, underscore, hyphen; leave blank for the unnamed default). PTTPlot saves the current configuration to a basename.{name}.pttp file in the same directory as the .pttd file. If a file with that name already exists, you are prompted to confirm overwrite. This item is disabled until a file is loaded.

Re-Plot On PTTD Change

A checkable menu item that controls whether PTTPlot automatically reloads and re-renders the chart when the .pttd file is modified by an external application (such as PTTEdit). When checked, the file watcher is active and live-preview mode is enabled. When unchecked, the file watcher is paused and the chart is not updated until the item is re-checked or the file is manually reopened. This item is disabled until a file is loaded.

Tip: Keep Re-Plot On PTTD Change checked while actively editing in PTTEdit to get an automatic live preview. Uncheck it temporarily if you need to prevent re-renders while making multiple rapid changes.

Name: {…}

A read-only label (shown as a disabled menu item) displaying the name of the currently loaded presentation in braces. For example, Name: {} indicates the unnamed default presentation is active; Name: {overview} indicates the overview presentation. This updates automatically when a different presentation is loaded.

Format Submenu

The Format submenu groups all visual formatting options. It contains three sub-items: Bars, Process Attributes…, and Dependency Arrows.

Format → Bars → Stacking

Controls how tasks are arranged vertically within each process row.

OptionBehaviour
Stacked All tasks belonging to the same process share the same horizontal row. If two tasks overlap in time, they are stacked vertically within that row (one above the other). This produces a compact layout.
Unstacked Every task gets its own dedicated horizontal row, regardless of time overlap. Process tasks are grouped together but never share a row. This produces a more spread-out layout where all tasks are individually legible.

Format → Bars → Labels

Controls the position and orientation of task name labels drawn inside the bars.

Horizontal Alignment

OptionBehaviour
Left Justified Task label text is anchored to the left edge of each bar.
Center Justified Task label text is centered horizontally within each bar.

Label Rotation

OptionAngleAppearance
Rotation Horizontal (0°) 0° Labels display flat and horizontal — easiest to read for long bars.
Rotation Slanted (45°) 45° Labels are angled at 45° — useful when bars are narrow.
Rotation Vertical (90°) 90° Labels are rotated 90° (reading bottom-to-top) — maximises use of narrow bars.

Format → Process Attributes…

Opens the Process Attributes dialog — a modeless (non-blocking) window that lets you customize the bar color of each process independently.

  • The dialog displays a two-column table: process name on the left, a colored swatch button on the right.
  • Click a swatch to open a standard color picker. Selecting a color immediately replots the chart with the new color applied.
  • Revert — Restores all process colors to the state they were in when the dialog was opened.
  • Close — Dismisses the dialog. Any color changes made survive in the active configuration and will be written to the .pttp file the next time you use Presentation → Save….
Note: The dialog remains open while you continue to work in the main window. It refreshes automatically when the chart is reloaded. This item is disabled until a file is loaded.

Format → Dependency Arrows

Controls whether dependency arrows are shown persistently for all tasks, and selects how the arrow endpoints are calculated. These settings complement the per-bar click gestures described in Click Controls.

ItemTypeDescription
Show Predecessors Checkable toggle When checked, predecessor arrows are drawn for every task in the chart at all times — equivalent to applying Shift+Ctrl+Left-Click globally.
Show Successors Checkable toggle When checked, successor arrows are drawn for every task in the chart at all times — equivalent to applying Shift+Ctrl+Right-Click globally.
Time Exclusive radio Time mode: Arrow endpoints are placed at the exact start or end time values referenced in the formula. An arrow reflects the precise moment one task's time value feeds into another.
Task Exclusive radio Task mode: Arrows connect the full extents of predecessor and successor task bars (end of source bar to start of destination bar), regardless of which time value is referenced in the formula.
Tip: Time mode is most informative for formula-driven timelines where you want to see exactly which time point one task depends on. Task mode gives a cleaner overview when you simply want to visualize the sequencing between tasks.

5Mouse Wheel Controls

Prerequisite: Mouse wheel zoom and pan are only active when the Zoom Controls toolbar is visible. Enable it via Help → Zoom Controls. When the toolbar is hidden, the scroll wheel has no effect on the chart.
Gesture Action
Wheel Up / Wheel Down Pan the chart up or down (vertically) by 10% of the currently visible range per scroll tick.
Shift + Wheel Pan the chart left or right (horizontally) by 10% of the currently visible range per scroll tick.
Ctrl + Wheel Up Zoom in, centered on the current cursor position.
Ctrl + Wheel Down Zoom out, centered on the current cursor position.

Panning Limits

Panning is clamped to the full data range. You cannot pan beyond the chart boundaries — once the edge of the data is reached, further panning in that direction stops.

Zoom Limits

Zooming out is clamped to the initial full view (the view that shows all data). You cannot zoom out beyond the full chart extent.

Restoring the Full View

After zooming in with Ctrl + Wheel, click the Home button in the Zoom Controls toolbar to restore the initial full-chart view instantly.

6Click Controls

PTTPlot supports several mouse click interactions directly on the chart canvas. Most gestures require clicking on a task bar (a rendered horizontal rectangle). Clicks on empty canvas areas are ignored, with the exception of the "toggle all" gestures noted below.

Task Info Annotations

Gesture Action
Right-Click on bar Toggle a task info annotation box showing Process, Task, Start, End, and Duration. Clicking the same bar again removes the annotation.
Alt + Right-Click on bar Toggle a task info annotation that additionally shows the formula expressions used to compute the start and end values, displayed as Start-ƒ and End-ƒ. Useful for debugging formula-driven timelines.

Dependency Arrows

Dependency arrows visualize relationships between tasks: which tasks feed into a given task (predecessors) and which tasks a given task feeds into (successors).

Gesture Action
Ctrl + Left-Click on bar Toggle predecessor arrows for the clicked task — draws arrows from the tasks that feed into this task.
Ctrl + Right-Click on bar Toggle successor arrows for the clicked task — draws arrows from this task out to the tasks it feeds.
Shift + Ctrl + Left-Click anywhere Toggle predecessor arrows for all tasks at once. If any predecessor arrows are currently visible, all are removed; otherwise predecessor arrows are shown for every task in the chart.
Shift + Ctrl + Right-Click anywhere Toggle successor arrows for all tasks at once. If any successor arrows are currently visible, all are removed; otherwise successor arrows are shown for every task in the chart.

Notes on Dependency Arrows

  • Arrows are rendered as filled arrowhead patches drawn over the chart bars (z-order above bars).
  • Arrow colour is a slightly darkened version of the source task bar's colour.
  • When the source and destination bars are adjacent in time on the same row, the arrow spans both bars with a larger arrowhead to indicate the direct dependency.
  • Predecessor and successor arrows are independent — both types can be displayed simultaneously on the same task bar.
  • The Shift+Ctrl "toggle all" gestures work anywhere on the canvas, not only on bars.

7Zoom Controls Toolbar

PTTPlot includes the standard matplotlib navigation toolbar. It is hidden by default to keep the interface clean. Toggle it on or off via Help → Zoom Controls.

Note: Mouse-wheel scroll zoom and pan are only active while the toolbar is visible. Hiding the toolbar also disables scroll-wheel interaction with the chart.

Toolbar Buttons

Button Action
Home Restore the initial full view — resets both axes to show all chart data. Use this after zooming in to return to the full overview.
Back Navigate backward through the zoom/pan history, restoring the previous view state.
Forward Navigate forward through the zoom/pan history, re-applying a view that was navigated away from via Back.
Zoom to Rectangle Click and drag on the canvas to define a rectangular region. The chart zooms in to show only that region. Click the button again to deactivate the tool.

Toggling the Toolbar

Help → Zoom Controls acts as a toggle. A checkmark next to the menu item indicates the toolbar is currently visible. Selecting it again hides the toolbar and disables wheel-based interaction.

8Auto-Reload (File Watcher)

PTTPlot monitors the currently loaded .pttd file for external modifications using Qt's QFileSystemWatcher. This enables a live-preview workflow between PTTEdit and PTTPlot.

How It Works

  1. When a file is loaded successfully, PTTPlot registers it with the file watcher and enables the Presentation → Re-Plot On PTTD Change menu item.
  2. If the file is modified externally (for example, saved from PTTEdit), PTTPlot receives a change notification.
  3. A 200 ms delay is applied before reloading. This accommodates editors that save by deleting the original file and recreating it — the delay allows the new file to be fully written before PTTPlot reads it.
  4. After the delay, PTTPlot reloads the file from disk and re-renders the chart.
  5. If the file was recreated (rather than modified in place), PTTPlot automatically re-registers the new file with the watcher so monitoring continues.

Enabling and Disabling the File Watcher

The Presentation → Re-Plot On PTTD Change menu item is a checkable toggle. When checked (the default after a successful file load), the file watcher is active. When unchecked, the watcher is paused and the chart will not update automatically until monitoring is re-enabled. If the .pttd file is modified while monitoring is paused, PTTPlot will reload it the next time monitoring is re-enabled.

Note: If the monitored file is deleted or becomes unavailable, PTTPlot automatically disables file monitoring and displays a warning. Open an existing .pttd file to restore monitoring.

Live Editing Workflow

Open the same .pttd file in both PTTEdit and PTTPlot simultaneously. Each time you save in PTTEdit, PTTPlot automatically updates the chart — no manual reload step required.

Note: The file watcher monitors only the currently loaded file. If you open a different file in PTTPlot, the watcher switches to monitor the new file.

11Configuration System

PTTPlot uses a 3-tier configuration system. Settings cascade from lowest to highest priority, with higher tiers overriding lower ones. Before loading each new file, configuration is reset to tiers 1 and 2 (the baseline), then tier 3 is applied on top of that baseline.

Tier Source Scope Priority
1 Hardcoded defaults in pttplot.py All files Lowest
2 pttplot.ini — user-level INI file (stored in the user config directory) All files Low
3 <basename>.{<name>}.pttp — per-file presentation config (same directory as .pttd) One file / presentation Highest

Time Unit and X-Axis Label

The horizontal axis label is controlled by the time_unit field stored in the file_metadata section of the .pttd file. This value is set in PTTEdit via the Time Unit combo box in the status bar.

  • If time_unit is blank (the default), the x-axis label reads Time.
  • If time_unit is set (e.g. ms), the x-axis label reads Time (ms).
Note: The time unit is a label only — PTTPlot does not perform any unit conversion or axis scaling. Files created before v0.4.0-dev that lack a file_metadata section are treated as having a blank unit, so their charts are unaffected.

Presentation File Naming

Each .pttd file can have multiple named presentation files, each stored as a separate .pttp file using a dot-qualifier naming convention: 

sample.{}.pttp           ; default (unnamed) presentation
sample.{overview}.pttp   ; presentation named "overview"
sample.{detail}.pttp     ; presentation named "detail"

When a .pttd file is first loaded and no matching .pttp file exists, PTTPlot automatically creates basename.{}.pttp from the current baseline configuration. Use Presentation → Save… to create additional named presentations.

Note: Curly braces ({ and }) are reserved for presentation qualifiers and cannot appear in .pttd filenames. PTTPlot will reject any attempt to open a .pttd file whose name contains braces.

Backup on Save

When saving a presentation file (Presentation → Save…) to a path that already exists, PTTPlot can automatically create a timestamped backup before overwriting it. Backup files follow the same naming scheme as PTTEdit: <filename>.YYYYMMDD-HHMMSS.bak, placed in a configurable subfolder relative to the .pttp file (which is always in the same directory as the .pttd file). 

project.{}.pttp                          ; the live file
ptt_backups/
    project.{}.pttp.20260402-102700.bak  ; earlier save
    project.{}.pttp.20260402-103415.bak  ; most recent backup
Note: PTTPlot auto-creates a default basename.{}.pttp when a .pttd file is first loaded and no presentation file exists. This auto-creation does not trigger a backup — backups are only made during explicit Presentation → Save… operations.

Backup behaviour is controlled by three keys in the [BACKUPS] section of pttplot.ini:

KeyDefaultDescription
backups_on_bool True Enable or disable backup on save. Set to False to turn off backups entirely.
backups_folder ptt_backups Subfolder name for backup files, relative to the .pttp file location. Spaces are allowed (quotes are optional). Leave blank to place backups in the same folder as the .pttp file.
backups_max_int 100 Maximum number of backup copies to keep. When the limit is reached, the oldest backup is deleted before a new one is written. Set to 0 or less for unlimited backups.

INI File Rules

All INI files consumed by PTTPlot follow these conventions:

  • Parser: configparser.RawConfigParser — no % interpolation, which preserves values like 25% as literal text.
  • Comment character: semicolon (;) only. The hash character (#) is not treated as a comment — this allows hex colour values such as #ff0000 to appear in INI values without any escaping.
  • Files are always read with encoding='utf-8' to avoid issues on Windows systems where the default encoding is cp1252.
; pttplot.ini example
; Use semicolons for comments -- # is NOT a comment character in this parser

[ANNOTATIONS.MARKER_DEFAULTS]
linestyle = dashed
color     = #2c5282    ; hex colors work correctly -- # is not a comment
fontsize_float  = 9
position  = 25%         ; % is preserved literally by RawConfigParser

[ANNOTATIONS.MARKERS]
marker1 = label=Project Start; time=0; linestyle=solid; linewidth_float=1.5;
 color=#276749; fontsize_float=10; fontstyle=italic; position=Top; rotation_float=90
marker2 = label=Milestone A; time=End(Process1:Task3); linestyle=dashed; linewidth_float=1;
 color=#c05621; fontsize_float=9; fontstyle=normal; position=25%; rotation_float=0

Annotation Markers

Markers are vertical reference lines with optional text labels. They are defined in the [ANNOTATIONS.MARKERS] section of any INI file. Markers from the global pttplot.ini and a per-file .pttp file are merged rather than replaced — a per-file .pttp can add new markers without removing the globally defined ones.

Marker Key Naming

Each marker key must start with marker followed immediately by a digit (e.g., marker1, marker2, marker10).

Marker Definition Format

Each marker is one INI line with 9 comma-separated fields:

markerN = label, time, linestyle, linewidth_float, color, fontsize_float, fontstyle, position, rotation_float
Field Type Description Accepted Values
label string Text displayed next to the marker line. Leave blank for a line with no label. Project Start, Go-Live, or empty
time number or formula Horizontal position of the marker, in the same time units as the chart data. Accepts either a numeric value or a task reference formula resolved at plot time:
  • time=5.0 — literal float
  • time=Start(ProcessName:TaskName)
  • time=End(ProcessName:TaskName)
Supported function names (case-sensitive): Start, StartTime, End, EndTime.
If the referenced Process:Task is not found in the plot data, the marker is silently skipped and a warning is logged. 		0, 14.5,
Start(Proc1:Task2),
End(Proc1:Task2)
linestyle string Style of the vertical line drawn at the marker position. solid, dashed, dotted, dashdot, None
linewidth_float number Width of the vertical line in points. Use 0 to hide the line while keeping the label visible. 1, 1.5, 0
color string Colour applied to both the line and the label text. Accepts any matplotlib colour string or hex value. #2c5282, red, tab:blue
fontsize_float number Font size of the label text in points. 9, 10, 11
fontstyle string Font style of the label text. normal, italic, bold
position string Vertical position of the label along the marker line. Top, Bottom, Center, 25% (percent from bottom)
rotation_float number Label text rotation angle. 0 = vertical, reading bottom-to-top. 90 = horizontal, reading left-to-right. 0, 45, 90
Blank fields: Any field left blank (two consecutive commas) inherits its value from the section-level default for that field.

Section-Level Defaults

Set defaults that apply to all markers in the section-level defaults. Individual marker fields that are left blank will inherit these defaults: 

[ANNOTATIONS.MARKER_DEFAULTS]
linestyle = dashed
linewidth_float = 1
color     = #555555
fontsize_float  = 9
fontstyle = normal
position  = Top
rotation_float  = 90

[ANNOTATIONS.MARKERS]
; marker2 overrides only color and position; all other fields use section defaults:
marker2 = label=Milestone B; time=45; color=#c05621; position=25%

Removing a Global Marker in a Per-File Presentation

To suppress a marker defined in the global pttplot.ini for a specific presentation, assign the marker key an empty value in the .pttp file: 

; Suppress marker1 that was defined in pttplot.ini for this presentation only:
marker1 =

Color Lists

Process bar colours are defined as a comma-separated list in the INI configuration. Each entry is validated against matplotlib's colour parser. If the list is empty or contains invalid colour values, PTTPlot falls back to matplotlib's tab20 colour map, which automatically assigns distinct colours to each process.
