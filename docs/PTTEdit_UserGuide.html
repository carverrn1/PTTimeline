1Overview

PTTEdit is the spreadsheet-like data editor for creating and editing .pttd timeline data files. It provides a formula system for defining task start and end times, including dependency relationships between tasks. The resulting data file is consumed by PTTPlot to render a Gantt-style horizontal bar chart, which can then be viewed or exported through PTTView.

The PTTimeline pipeline works as follows:

  1. PTTEdit — Create or edit a .pttd data file containing process and task definitions with time formulas.
  2. PTTPlot — Open the .pttd file to render and export the Gantt-style chart.
  3. PTTView — View the exported image file (SVG, PNG, JPG, or PDF).

PTTEdit is built with Python 3.10, PySide6 (Qt GUI framework), and pandas/numpy for data handling. Data files are stored in JSON5 format (JSON extended with comment support) using the .pttd extension.

2Command-Line Usage

Launch PTTEdit from a terminal with an optional filename argument:

python pttedit.py [filename.pttd]
  • If a filename is provided, that file is opened immediately on startup.
  • The .pttd extension is appended automatically if the argument has no extension.
  • If no filename is given, PTTEdit starts with a blank interface containing one empty row.
Note: PTTEdit must be run from the directory containing pttedit.py, or from a location where the lib/ subdirectory is accessible relative to the script. Support modules are resolved from that subdirectory at startup.

3The Grid (Spreadsheet)

The central element of PTTEdit is a spreadsheet-style grid. Each row represents one task in the timeline. There are seven columns: four that are directly editable and three that are read-only (automatically computed from the formulas).

Columns

# Header Editable Internal Name Description
0 Process Yes ProcessName Identifier for a process (a named row group). Must begin with a letter or underscore, followed by letters, digits, or underscores only.
NOTE: The ProcessName may be left blank (i.e unnamed). This is a special case to allow pseudo-constants to be defined in the table, used in cell formulas, but not plotted. The TaskName is used as the name of the constant and referenced as ':task'in formulas (e.g. Value(:MOTOR1_MAX_TIME).
1 Task Yes TaskName Identifier for a task within a process. Same character rules as ProcessName. The combination ProcessName:TaskName must be unique across all rows in the file.
2 Start-ƒ Yes StartTimeFormula Formula expression that defines the task start time. Plain numbers are valid (e.g., 0, 1.5). May reference other tasks.
3 Start No (calculated) StartTime Evaluated numeric result of StartTimeFormula. Displayed in italic grey. Automatically recomputed whenever any formula in the grid changes.
4 End-ƒ Yes EndTimeFormula Formula expression that defines the task end time. Same syntax rules as StartTimeFormula.
5 End No (calculated) EndTime Evaluated numeric result of EndTimeFormula. Displayed in italic grey.
6 Duration No (calculated) Duration Computed as EndTime − StartTime. Displayed in italic grey. A negative value indicates the end time is earlier than the start time.

Read-only columns (Start, End, Duration) use a grey background and italic text to visually distinguish them from editable columns. Clicking on these cells does not open an editor; their values are derived entirely from the formula columns.

Cell Highlighting

PTTEdit uses background and text color to communicate calculation status at a glance without requiring the user to inspect individual cells:

Condition Appearance Example
Normal calculated value Grey background, italic dark grey text 3.5
Formula evaluation error Grey background, red bold text (error code shown) EXPR_ERR
Negative duration Light yellow background, dark red bold text −2.0
Warning: A negative duration means a task's end time is earlier than its start time. PTTPlot may render such tasks incorrectly or omit them. Resolve negative durations before generating a chart.

4File Operations

All file operations are accessible from the File menu.

Command Shortcut Description
New — Clear the grid and create an empty timeline containing one blank row. Prompts to save unsaved changes before clearing.
Open Ctrl+O Open a .pttd file via a file picker dialog. The file is parsed as JSON5. Prompts to save unsaved changes before opening.
Open Recent — Opens a submenu listing the most recently opened .pttd files (up to 15 entries, newest first). Entries are numbered 1–9 for quick keyboard access. Hovering over any entry shows the full path in a tooltip. Stale entries (files no longer on disk) are silently removed when the submenu is opened. A Clear Recent Files item at the bottom clears the entire list. The file dialog for Open and Save As defaults to the directory of the most recent entry, falling back to the OS default when the list is empty.
Save Ctrl+S Save the current file to its existing path. If the file has no filename yet (never been saved), behaves as Save As.
Save As… — Save the current data to a new filename chosen via a file picker dialog. The .pttd extension is appended automatically if omitted.
Import CSV — Replace the current timeline with data from a CSV file. The CSV must contain these columns in order: ProcessName, TaskName, StartTimeFormula, StartTime, EndTimeFormula, EndTime, Duration. Formula strings are preserved; computed columns are recalculated after import. Prompts to save unsaved changes before replacing.
Append CSV — Append rows from a CSV file to the end of the current timeline without discarding existing data. The CSV must use the same column layout as Import CSV. Duplicate ProcessName:TaskName pairs are automatically renamed using the _COPY_## suffix to avoid conflicts. Prompts to save unsaved changes before appending.
Export CSV — Export the current timeline to a CSV file. The output path is derived automatically from the current .pttd filename (same directory and base name, with a .csv extension). If a file with that name already exists, a confirmation dialog is shown before overwriting. This item is disabled until a file has been loaded or saved.
Export ODS — Export the current timeline to an OpenDocument Spreadsheet (.ods) file. The output path follows the same convention as Export CSV. This item is disabled until a file has been loaded or saved.
Export UML Timing Diagram — Export the current timeline as a PlantUML timing diagram source file (.puml). The output path is derived automatically from the current .pttd filename (same directory and base name, with a .puml extension). If a file with that name already exists, a confirmation dialog is shown before overwriting. This item is disabled until a file has been loaded or saved. If a time unit is set, it is included as a comment in the .puml header (e.g. ' time_unit: ms).
Note: PTTEdit generates only the .puml source file. Rendering the diagram image requires PlantUML (plantuml.jar) and a Java Runtime Environment (JRE) to be installed separately on your system. Once installed, open the .puml file with PlantUML to generate the PNG diagram.
Exit — Close the application. If there are unsaved changes, a Save / Discard / Cancel dialog is shown before exiting.

Backup on Save

When saving to a file that already exists, PTTEdit can automatically create a timestamped backup before overwriting it. This protects against accidental data loss and lets you revert to a previous save if needed.

Backup files are named using the format <filename>.YYYYMMDD-HHMMSS.bak and are placed in a configurable subfolder relative to the .pttd file: 

project.pttd                          ; the live file
ptt_backups/
    project.pttd.20260402-102700.bak  ; earlier save
    project.pttd.20260402-103415.bak  ; most recent backup

No backup is created on the very first save (when the file does not yet exist). The backup folder is created automatically if it does not exist. If the backup folder is left blank, backups are placed in the same directory as the .pttd file.

When the number of existing backups reaches backups_max_int, the oldest backup is deleted before the new one is written. The oldest backup is determined by the timestamp embedded in the filename, so the ordering is reliable regardless of OS file timestamps. Set backups_max_int to 0 or less for unlimited backups.

Backup behaviour is controlled by three keys in the [BACKUPS] section of pttedit.ini:

KeyDefaultDescription
backups_on_bool True Enable or disable backup on save. Set to False to turn off backups entirely.
backups_folder ptt_backups Subfolder name for backup files, relative to the .pttd file location. Spaces are allowed (quotes are optional). Leave blank to place backups in the same folder as the .pttd file.
backups_max_int 100 Maximum number of backup copies to keep. When the limit is reached, the oldest backup is deleted before a new one is written. Set to 0 or less for unlimited backups.

File Format (.pttd)

PTTEdit data files use JSON5 format (JSON extended with comment and trailing-comma support) and carry the .pttd extension. A .pttd file contains three top-level sections:

KeyContents
file_metadata File-level metadata describing the data, including time_unit (the time unit label set in the status bar). This section is read by both PTTEdit and PTTPlot.
config Application configuration settings saved with the file (DEBUGGING, EXTERNAL_PROGRAMS, etc.).
dataframe Row data for every task, including the raw formula strings (StartTimeFormula, EndTimeFormula) and the last evaluated numeric results (StartTime, EndTime, Duration).

Because JSON5 is used, you may add // line comments or /* */ block comments when editing .pttd files manually in a text editor. Be aware that PTTEdit's own Save function does not preserve comments; manual comments will be removed when the file is saved by PTTEdit.

Backward compatibility: Files created before v0.4.0-dev do not contain a file_metadata section. Both PTTEdit and PTTPlot treat the absence of this section as equivalent to time_unit = "" (no unit label), preserving existing behaviour.
Note: .pttd files are plain text and are suitable for version control with any standard VCS (git, SVN, etc.).

Unsaved Changes

When the current timeline contains unsaved changes, PTTEdit appends an asterisk to the window title: PTTEdit — myproject.pttd *.

Attempting to open a new file, create a new timeline, or exit while unsaved changes exist triggers a prompt with three options:

  • Save — Save the current file and then continue the requested operation.
  • Discard — Abandon unsaved changes and continue without saving.
  • Cancel — Return to the editor without performing any operation.

5The Formula System

Formulas are entered in the Start-ƒ and End-ƒ columns. A formula is any expression that evaluates to a numeric value. A plain number is a valid formula (e.g., 0, 1.5). Formulas may reference the computed start time, end time, or duration of any other task in the file.

Arithmetic Operators

Standard arithmetic operators are supported with conventional operator precedence: * and / are evaluated before + and -. Parentheses can override precedence.

OperatorOperationExample
+ Addition End($:Design) + 1
- Subtraction End($:Build) - 0.5
* Multiplication Duration($:Phase1) * 2
/ Division Duration($:Phase1) / 3

Reference Functions

Reference functions look up a computed value from another task. The target task is identified by a ProcessName:TaskName pair separated by a colon.

Function Aliases Returns
Start(P:T) none The evaluated Start time of task T in process P
End(P:T) none The evaluated End time of task T in process P
Duration(P:T) Value(P:T) The evaluated Duration time of task T in process P

All function names are case-sensitive.

Multi-Argument Functions

Multi-argument functions accept two or more comma-separated arguments. Each argument may itself be an arithmetic expression, a reference function call, or another multi-argument function call.

Function Arguments Returns
Min(a, b, …) Two or more values The minimum of all arguments
Max(a, b, …) Two or more values The maximum of all arguments
If(cond, a, b) Condition, true-value, false-value a if cond ≠ 0, otherwise b
Not(a) One value 1.0 if a = 0, otherwise 0.0
Or(a, b, …) Two or more values 1.0 if any argument ≠ 0, otherwise 0.0
And(a, b, …) Two or more values 1.0 if all arguments ≠ 0, otherwise 0.0
IsLessThan(a, b) Two values 1.0 if a < b, otherwise 0.0
IsGreaterThan(a, b) Two values 1.0 if a > b, otherwise 0.0
IsLessEqual(a, b) Two values 1.0 if a ≤ b, otherwise 0.0
IsGreaterEqual(a, b) Two values 1.0 if a ≥ b, otherwise 0.0
IsEqual(a, b) Two values 1.0 if a = b, otherwise 0.0
IsNotEqual(a, b) Two values 1.0 if a ≠ b, otherwise 0.0
Tip: Comparison functions return 1.0 (true) or 0.0 (false), making them directly composable with If(), And(), Or(), and Not() for building conditional logic.

$ Macro Expansion

The $ symbol is a positional macro that expands to the ProcessName or TaskName of the row containing the formula, depending on where it appears within a reference function argument.

Inside a FunctionName(P:T) argument, $ in the P position expands to the current row's ProcessName; $ in the T position expands to the current row's TaskName.

Formula Written Row (Process : Task) Expands To
Start($:Task1) Process1 : Task2 Start(Process1:Task1)
End(Process1:$) Process1 : Task2 End(Process1:Task2)
Start($:$) Process1 : Task2 Start(Process1:Task2)

The $ macro is especially useful when copying rows or designing formulas that should always reference the current process without hard-coding its name, making rows more portable when process names change.

Formula Examples

# Task starts at time 0
StartTimeFormula: 0

# Task ends 2.5 units after the Design task in the same process starts
EndTimeFormula:   Start($:Design) + 2.5

# Task starts immediately when its predecessor ends
StartTimeFormula: End($:Design)

# Task starts at the latest end time of two predecessor tasks
StartTimeFormula: Max(End(TeamA:Design), End(TeamB:Prototype))

# Task ends at the midpoint between two tasks
EndTimeFormula:   (End($:PhaseA) + Start($:PhaseB)) / 2

# Start at 5 if Design duration exceeds 3 units, otherwise start at 3
StartTimeFormula: If(IsGreaterThan(Duration($:Design), 3), 5, 3)

# Start only after both parallel branches complete
StartTimeFormula: Max(End(Branch1:Build), End(Branch2:Build))

Circular Reference Detection

PTTEdit detects circular dependencies in the formula graph before evaluation. A circular dependency exists when task A references task B, and task B references task A, whether directly or through a chain of intermediate tasks.

The expression evaluator uses topological sorting (Kahn's algorithm) to determine the correct evaluation order. Tasks with no dependencies are evaluated first; tasks depending on others are evaluated after their dependencies resolve. If the dependency graph contains a cycle, all tasks involved are flagged with CIRC_ERR and cannot produce a value until the cycle is broken.

Warning: Circular references must be resolved before PTTPlot can be launched. Break the cycle by removing or redirecting at least one formula that participates in the cycle.

Error Codes

When a cell cannot be evaluated, an error code is displayed in the calculated column. Error cells are shown with red bold text. The status bar also displays the total count of cells containing errors.

Error CodeMeaning
EXPR_ERR The formula string contains invalid expression syntax. Check for unmatched parentheses, unknown function names, or typographical errors.
STRT_ERR The formula parsed successfully but an error occurred while evaluating the start time (for example, a referenced task does not exist).
END_ERR The formula parsed successfully but an error occurred while evaluating the end time.
DRTN_ERR An error occurred while computing Duration from the evaluated start and end times.
CIRC_ERR A circular dependency was detected. This task is part of a reference cycle and cannot be evaluated until the cycle is broken.
Note: Launching PTTPlot is blocked while any cell in the grid displays an error code (any value ending in ERR). Resolve all errors first.

6Row Operations

Row operations are available via the Edit menu or by right-clicking on a row number in the vertical header on the left side of the grid. A row must be selected before these operations apply.

OperationDescription
Move Row Up Move the selected row one position upward in the grid. This affects the display and export order only; formula references use ProcessName:TaskName identifiers and are not affected by row position.
Move Row Down Move the selected row one position downward in the grid.
Add Row Above Insert a new blank row immediately above the selected row.
Add Row Below Insert a new blank row immediately below the selected row.
Copy Row Copy the selected row to an internal clipboard, storing ProcessName, TaskName, and both formula strings.
Paste Row Insert the copied row below the currently selected row. If the resulting ProcessName:TaskName combination would duplicate an existing row, the TaskName is automatically renamed with an incrementing suffix to avoid the conflict. For example, if Task1 already exists in the same process, the pasted TaskName becomes Task1_COPY_1; if that also exists, Task1_COPY_2, and so on. The ProcessName is copied as-is and may be blank (blank ProcessName rows are valid).
Delete Row Delete the selected row after displaying a confirmation dialog. This action cannot be undone.
Tip: Row order in the .pttd file determines the display order of processes and tasks in the PTTPlot chart. Use Move Row Up / Down to arrange tasks in the desired visual sequence before rendering.

7Context Menus

PTTEdit provides two context menus accessible by right-clicking within the grid.

Row Number Context Menu

Right-click on a row number (the numeric label in the vertical header at the left edge of the grid) to open the row operations menu. This menu provides all row-level operations:

  • Move Row Up
  • Move Row Down
  • Add Row Above
  • Add Row Below
  • Copy Row
  • Paste Row
  • Delete Row

Process Cell Context Menu

Right-click on a cell in the Process column to open a dropdown list of all ProcessName values that currently exist in the timeline. Selecting a name from this list populates the cell with that ProcessName, making it easy to assign a task to an existing process without typing.

Tip: The Process cell context menu is the fastest way to add many tasks to the same process. It eliminates typing errors that would create duplicate or misspelled process names.

8Edit Menu (Find & Rename)

The Edit menu provides Find and Rename commands for locating and bulk-renaming text across the timeline grid.

Find  Ctrl+F

Opens the Find dialog, which searches the Process, Task, Start-ƒ, and End-ƒ columns for literal text.

ControlDescription
Search text The literal string to search for. Search is performed against formula text, not evaluated values.
Case Sensitive When checked, the search distinguishes uppercase from lowercase.
Selected Rows Limits the search to a subset of rows. Accepts flexible range syntax: 1-N (all rows), 3-10, 1,2,5, 1-20,30,50-60, 5- (row 5 to end). Defaults to 1-N.
Find Next / Find Previous Navigate matches one at a time with wrap-around. The status line shows the match count.
Find All Opens the Find All Results window listing every match.

Find All Results Window

The Find All Results window is modeless (stays open while you edit). Each result is displayed in Process:Task.column: formula format with the matched text highlighted in yellow. Click any result to navigate the grid to that cell; the clicked result grays its background as a visual indicator. A Refresh button re-runs the search against the current grid contents.

Rename  Ctrl+R

Opens the Rename dialog for atomically renaming a Process Name or a Process:Task Name across the entire file without breaking existing formula references.

ControlDescription
Scope Process Names — renames cells in the Process column and updates the process-name portion of all Start-ƒ/End-ƒ formula tokens.
Process:Task Names — renames the definition row and all formula references; also handles blank-process variable references (e.g. :VarName).
From / To The current name and the desired replacement name. Both are validated against the naming rules (alphanumeric and underscore only for Process/Task names) with clear inline error messages.
Selected Rows Limits the rename to a subset of rows using the same range syntax as Find. Defaults to 1-N (all rows).
Preview All Opens the Rename Preview window showing every cell that will change before committing.
Rename All Commits all changes immediately without previewing.

Rename Preview Window

The Rename Preview window lists every cell that will change. The original text is highlighted in yellow and the replacement text in light blue. Click any result row to navigate the grid to that cell; both the before and after lines gray on click for visual feedback. A Rename All button at the bottom commits the changes directly from the preview without returning to the Rename dialog.

Note: Rename updates are atomic — all affected cells are changed in a single operation. The file is marked modified after a rename; save to make the changes permanent.

10Launching PTTPlot

View → Plot

The View → Plot menu item launches PTTPlot with the currently loaded .pttd file as its argument. PTTPlot renders the timeline as a Gantt-style horizontal bar chart and opens in its own window.

Prerequisites

  • The current file must be saved. PTTEdit will prompt to save before launching if there are unsaved changes.
  • The timeline must have no formula errors. If any cell displays an error code, the Plot command is disabled until all errors are resolved.

How PTTPlot is Located

PTTEdit searches for the PTTPlot executable in the following order:

  1. PTTPlot.exe in the same directory as pttedit.py (for packaged or frozen distributions).
  2. python pttplot.py using the Python interpreter, with pttplot.py in the same directory (for source installations).
  3. A path explicitly configured in pttedit.ini under the [EXTERNAL_PROGRAMS] section (for custom or non-standard setups).

Only one PTTPlot instance can be open at a time from PTTEdit. If a PTTPlot window is already open, a message is displayed (“Close plotter to open a new plot”) and a new instance is not launched. Close the existing PTTPlot window first to plot a different file.

Note: PTTPlot includes a built-in file watcher that monitors the .pttd file for external changes. After you save changes in PTTEdit, PTTPlot automatically reloads the file and refreshes the chart without requiring a manual reload.

11Status Bar

The status bar runs along the bottom edge of the PTTEdit window and provides real-time feedback. The left side shows status messages; the right side holds the Time Unit selector and the error indicator:

ZoneContent
Left Status messages describing the current operation, such as: Ready, Saving…, Recalculating…, File loaded, Import complete.
Right — Time Unit An editable combo box labelled Time Unit: that sets the time unit label for the current file. Select a unit from the dropdown list or type any custom label directly into the field. Leave blank for no unit label (the default for new files).
Note: The time unit is stored as file-level metadata in the .pttd file and is used by PTTPlot to annotate the horizontal axis. Changing the value marks the file as modified.
Far Right — Errors Error summary indicator. Shows No errors in green when the grid is valid, or Errors: N in red where N is the count of cells containing an error code. This indicator updates automatically on every recalculation.
